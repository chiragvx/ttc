# THE PLAYBOOK — Building Grounded Text-to-CAD With Claude

## 1. The Strategic Call

Build the **conversational parametric generator for functional printable/machinable parts** — brackets, motor/sensor mounts, electronics enclosures, manufacturing fixtures and jigs — with real DFM, a real slicer-backed cost/time number, and a real Gmsh+CalculiX linear-static Factor-of-Safety under one declared load. This wedge exercises *every* load-bearing thesis (LLM-emits-validated-deltas-not-code, deterministic Jinja2+build123d templating, persistent topological identity, bounded-slider two-plane UX, grounded-solver safety numbers) while shedding the entire un-shippable aerospace burden (flutter/DLM, AVL/XFOIL aero, propulsion/range, kinematics, bonded-joint cohesive FEA, ITAR, AS9100); **aerospace is the Series A/B narrative, matured behind the wedge, never the beachhead.** Claude changes the economics because the product's own keystone inversion is *also* the dev-time operating model — the same typed seams (hardened ledger schema, `ParameterDef`, `BaseSolverDriver`, feature-template contracts) that make the runtime safe are exactly what let parallel Claude Code subagents build the system without colliding, compressing a conventional 12-15 person team to ~5-6.

Two are STOP-the-company gates (identity, solver), one is a scope-STOP (codegen vocabulary), two degrade-not-stop.

## 3. Phases 1-3

### Phase 1 — Foundation (30-90 days)
**Goal:** a hardened, deterministic, single-tenant substrate built *only on what survived the spikes.*
**Build:**
- **Executable ledger schema as the spec first:** Pydantic v2 with one reusable `ParameterDef` (`{value, unit, lock_state, bounds[min,max], precision}`) referenced by every tunable node, `additionalProperties:false` everywhere, invariants as validators (`bounds[0]≤value≤bounds[1]`, `skin≥min_wall`, HARD_LOCK immune to optimization). Every runtime-written node (incl. cascade targets) defined or a test fails. **This same artifact is the runtime tool-use schema.**
- Postgres **FACTS-vs-DERIVATIONS** event store with actor-id, `prev_event_hash` chaining, `review_state`, toolchain fingerprint + external-dataset content-hashes.
- **Production** generator+tag layer (rebuilt clean from spike lessons) with the post-compile OCC round-trip verifier.
- Firecracker/gVisor warm sandbox pool; delta→template codegen on Sonnet tool-use; **exactly ONE solver** (CalculiX FS) wired as an async job.
- **No optimizer, no flutter, no UI beyond a dev harness, no multi-tenancy.**

**How Claude is used:** Spec-driven dev — Claude (plan mode) drafts ledger schema + gate tests from the spec, you review the *plan* before code. `BaseSolverDriver` + `add-solver-driver` skill + `add-feature-template` skill encode repeated patterns. Hooks (`.claude/settings.json`): PostToolUse runs pyright/ruff/schema-tests; pre-commit runs the round-trip verifier + golden-artifact determinism test; a hook rejects non-allowlisted imports in generated templates. Per-module `CLAUDE.md` with hard prohibitions; scoped subagents per module.
**Proof-point gating Phase 2:** the hero bracket renders through conversation→delta→templater→sandboxed regen→analytic HUD→export, with a real CalculiX FS landing as a DERIVATION event, and a green determinism/round-trip suite (HARD_LOCK 4.5mm pin never drifts to 4.50001mm). **This slice is simultaneously the investor demo and the production foundation.**

### Phase 2 — MVP (3-6 mo)
**Goal:** a real, grounded, single-tenant product with human-in-the-loop sign-off — "it printed, it fit, the predicted strength was right."
**Build (in dependency order that unblocks safety gates, NOT demo-appeal order):**
1. **Requirements/verification-traceability matrix first** (requirement-ID → parameters → verification method → margin) — it's both a missing subsystem and the executable-spec backbone.
2. Positioned BOM + material-property DB + datum/reference frame → makes mass/CG computable.
3. Real PrusaSlicer sidecar (cost/time) + the FEA convergence/cross-validation tier (Euler-Bernoulli cantilever, Roark/Euler buckling golden cases; FDM-anisotropy knockdowns; `unknown` blocks export).
4. **Propose→review→commit FSM**: every delta lands as `ai_proposed`; only an explicit engineer accept writes the committing `engineer_reviewed` event. Geometry merges never auto-applied; last-writer-wins on geometry forbidden.
5. **ONLY THEN** the React/r3f split-pane viewport, morph preview, anchored HUD (bound to proven feature tags), two-plane WebSocket protocol.

**How Claude is used:** `/workflows` fan-out — the ~8 missing subsystems share the same `schema-domain + analytic-proxy + driver + tests` anatomy; one subagent each against shared contracts. Golden-conversation evals (intent→delta accuracy + ambiguity/`request_clarification` for "6S"/"2 inch"/"make it stronger") run in CI; Batches API for cheap sweeps. Reuse spike-era prompt-injection + mesh-convergence evals as permanent regression gates.
**Proof-point gating Phase 3:** **PRINTS-AND-FITS + PASSES-A-REAL-LOAD-TEST** — a generated bracket prints, mates within DFM tolerance, is costed, and its predicted FS correlates to physical failure on a design partner's load frame ("predicted 1.7×, broke at 1.65×"). That correlation is the moat and the only credible on-ramp to aerospace.

### Phase 3 — Scale / Aerospace (6-12 mo)
**Goal:** multi-tenant, audit-ready, controlled-data-capable; aerospace physics matured behind the proven wedge.
**Build:** Postgres RLS multi-tenancy; Temporal durable solver DAGs; KEDA autoscale; the air-gapped vLLM ITAR SKU; AS9100/DFARS audit substrate hardened; AGPL-slicer isolation + OCCT-LGPL legal sign-off (gated milestone, §6). Surrogate training (GP/POD with validity domains + error bars) + a real NSGA-II/qEHVI optimizer with buckling/modal/flutter as constraint-domination — **only now**, because only now does a labeled solver corpus exist. Then the aerospace subsystems (AVL/XFOIL, p-k DLM flutter, bonded-joint cohesive FEA, propulsion/range, kinematics/swept-volume).
**How Claude is used:** the `LLMProvider` seam now serves vLLM (Qwen2.5-Coder/Llama-3.x-70B) with **xgrammar/outlines guided decoding generated from the same `ParameterDelta` schema**; the existing eval suite is the air-gapped SKU's acceptance bar. Scheduled Claude subagent (cron via MCP) watches upstream OCCT/solver releases, runs the golden suite against bumps, and files **explained** rebaseline PRs. Claude generates AS9100 audit narratives as a **read-only projection** over the deterministic event store — summarizes stored facts, never recomputes them.

## 4. The Claude Operating Model

### (a) DEV-TIME — Claude Code engineers the product
- **Spec-driven dev:** the hardened Pydantic ledger schema + Jinja2 templates ARE the spec; Claude generates models, Alembic migrations, the round-trip verifier, and tests from it, in plan mode for anything touching the safety-numeric or identity path (you review approach before code).
- **Repo mirrors the two planes:** `ledger/`, `interactive_plane/` (analytic proxies, closed-form only), `truth_plane/regen/` (templater+build123d worker), `truth_plane/solvers/` (driver fleet), `agents/`, `transport/`, `frontend/`. **The only thing crossing any boundary is a Pydantic model** — no shared mutable state, no free dicts. Dense per-module `CLAUDE.md` states the contract + hard prohibitions (e.g. `interactive_plane/CLAUDE.md`: "no OCCT, no LLM, no solver, closed-form arithmetic only, <1ms/number"). Root `CLAUDE.md` encodes the three inversions verbatim **and the cut list under "DO NOT BUILD (yet)"** so subagents don't gold-plate the PRD fantasy (no Kubernetes/Temporal/surrogates/Rust core in the wedge).
- **Parallel subagents** scoped to single modules in isolated worktrees — never one mega-agent over the whole tree (that hallucinates cross-layer coupling and reintroduces the single-clock fallacy).
- **Custom skills:** `add-feature-template`, `add-solver-driver`. **`/workflows`** fan out the mechanical sweeps (solver drivers one-per-binary, the ~8 missing subsystems).
- **MCP servers:** one wrapping the build123d kernel + solver CLIs (Claude self-tests geometry against real OCC bbox/topology output); an **OCP-introspection MCP** exposing the installed wheel's real symbol table so Claude grounds OCAF/TNaming code against actual signatures.
- **Verification-gated loop:** hooks make type-check + schema-test + round-trip + determinism-fingerprint **unskippable**; CI re-runs as the merge gate.
- **Where Claude excels:** Python/FastAPI/Pydantic, Jinja2, build123d feature scripts, solver CLI wrappers + output parsers, React/r3f, Temporal workflows, test generation — let it run.
- **Where it needs a human harness (three walled zones):** (1) OCAF/TNaming + OCP FFI — Claude writes fluent code against *phantom* pybind11 symbols; require OCP-introspection MCP + plan mode + a naming-stability property test, this is the contained escalation point to a C++/Rust OCCT worker owned by a specialist. (2) Numerical-determinism debugging — Claude can't feel BLAS-thread/FMA/rounding divergence; pin truth on the canonical image. (3) FEA physics correctness — Claude writes drivers that return plausible-but-wrong numbers (yield-vs-buckling, unconverged mesh); golden values come from closed-form/handbook/PE, **never Claude's own run.**

### (b) RUNTIME — Claude powers the product
- **Role × model × clock matrix, enforced as routing not prompt convention:**
  - **Opus 4.8** (`claude-opus-4-8`, adaptive thinking, effort high/xhigh) = Strategic Systems Engineer on the **human-timescale conversational turn** (mission→requirements, branch comparison, disambiguation). Allowed to be slow.
  - **Sonnet 4.6** (`claude-sonnet-4-6`, effort low) = geometric **delta-emitter** on slider-release.
  - **Haiku 4.5** (`claude-haiku-4-5`) = cheap classification (units guess, intent routing).
  - **Validator = NOT a model.** A deterministic rules/solver router ingesting Gmsh+CalculiX/AVL/XFOIL/p-k output → FS/stall/flutter/range. Add a static check asserting no model-call path can write any node flagged `derived_safety_scalar`.
- **Tool-use contract = the only legal emission:** `propose_parameter_delta` Pydantic model compiled to strict JSON Schema (`additionalProperties:false`, `$defs/ParameterDef`), enforced with `tool_choice:{type:"tool",name:"propose_parameter_delta"}` + `strict:true` — prose/free Python is impossible at the API layer. Schema enforces SHAPE; the rules router enforces INVARIANTS (bounds clamp, HARD_LOCK, coupled invariants → CONFLICT state). `request_clarification` is a first-class tool the agent is rewarded for choosing on ambiguity.
- **Manual tool-use loop, never the auto-running `tool_runner`:** the tool call is an AI-PROPOSED diff; a separate explicit human accept commits — preserving the sign-off gate AS9100/traceability/liability require.
- **Grounding:** pgvector RAG over requirements matrix + material/component DB; `lookup_material`/`lookup_requirement` return `(row_id, value, content_hash)` injected into the cached prefix. **Cited values flow to solvers/HUD, never re-spoken as authoritative LLM numbers.**
- **Evals:** golden-conversation suite grades the structured **delta (the FACT), never prose**; pin the model snapshot id; 100% required on the "never emit a safety scalar / never violate HARD_LOCK" subset; clarification-precision + over-confident-guess rate as release gates. LLM-as-judge only for semantic equivalence of two structured deltas — never for FS/watertightness/any export gate.
- **Cost control:** freeze schema/system/tools as a `cache_control:{type:"ephemeral"}` prefix (~0.1× reads); inject live solver verdicts + HUD snapshots as **mid-conversation `{"role":"system"}` messages** (Opus 4.8, no beta header) — **never** at the front of the system prompt (a single `datetime.now()` or per-design UUID there drops cache hit-rate to zero and ~10×'s cost/design). Log `usage` per design as ledger events; `count_tokens` pre-flight; Batches API (50%) for non-interactive work. Don't chase reproducibility via sampling tricks — Opus 4.8/Sonnet 4.6 reject `temperature`/`top_p`; determinism lives downstream in the templater + content-addressed DERIVATION artifacts.
- **Air-gapped path:** one `LLMProvider` interface, hosted Anthropic vs vLLM swappable behind the identical schema/validator/evals; controlled-tenant traces/eval-samples/audit-summaries run **inside the enclave on open weights** (they are themselves controlled data). Handle `stop_reason:"refusal"` on Opus (defense/UAV-adjacent intent can trip safety classifiers) — surface, don't crash.

## 5. The Verification & Safety Harness (Day One)

Mirror the product thesis in the dev process: **never let a green light — a passing test or a perf claim — be fabricated.**

1. **Golden-geometry regression under a pinned toolchain fingerprint** (first CI artifact). Fingerprint = SHA-256 over OCCT/build123d/OCP wheel hashes + Python build + `OMP_NUM_THREADS=1` + FMA/rounding flags. Canonicalize STEP/B-rep (strip FILE_NAME timestamps, normalize OCCT entity ordering), tessellate at fixed deflection, store golden hashes of canonical B-rep AND mesh. Start with one deliberately-failing test that renders the 4.5mm pin twice and asserts identical mesh SHA-256 — surfaces every non-determinism source before anything builds on top.
2. **Replay purity** as a CI invariant: a pure fold over FACTS that rehydrates DERIVATIONS by hash and **never invokes LLM/optimizer/solver.** Inject `llm_tripwire`/`solver_tripwire` fakes that fail on any call; assert call-count==0 and DERIVATION hash-equality. Fingerprint-mismatch → serve stored artifact + raise flag, never silently recompute.
3. **Property-based geometry invariants (Hypothesis)** bound to `ParameterDef`: BRepCheck valid, watertight + 2-manifold, min-wall, no self-intersection, HARD_LOCK round-trip (locked 4.5mm reads back 4.5±1e-3 across 1000 examples). `RuleBasedStateMachine` over `{set, lock, cascade, merge}` sequences. **The invariant module IS the production round-trip verifier** — one source of truth for test and live gate. Keep a *separate* adversarial suite (out-of-bounds, HARD_LOCK-violating, poisoned `.dat`) that proves the validator FAILS CLOSED.
4. **Solver cross-validation + mesh-convergence gates:** every reported FS gated on a convergence study (FAIL if a PASS ever emitted on an unconverged mesh); FDM-anisotropy knockdowns verified applied; the `unknown`-blocks-export path tested. Reference coefficients cited from handbook + PE-verified, never Claude's word.
5. **Eval the delta not the prose** (pinned snapshot, schema-validation as the fail-closed gate); ambiguity/refusal evals required.
6. **Human-sign-off FSM** (`AI-PROPOSED → REVIEWED → COMMITTED → EXPORT-ELIGIBLE`) tested via model-based/Hypothesis-stateful tests: no geometry-class change reaches COMMITTED without a sign-off FACT; export blocked unless every gate green AND sign-off exists AND no solver result is `unknown`; CRDT merge violating bounds/HARD_LOCK → CONFLICT, never silent convergence.
7. **Hermetic CI/CD:** one container pins OCCT + all solvers by sha256; the test image **is** the prod worker image. Sandbox tested as a security surface: degenerate boolean SIGKILLed within budget, egress fails closed, RLIMIT caps hold, microVM crash never takes the host. **Golden-hash rebaselines are never auto-merged** — require a root-cause diff (which faces/dims moved, why) + human/PE sign-off.

**Hard anti-pattern across all of it:** never let Claude make a failing golden/property/solver test pass by loosening a tolerance, widening a bound, or weakening an assertion. A drifted hash or failed convergence is SIGNAL. Treat any such PR as a red flag requiring human sign-off. The LLM is never the test oracle for geometry validity or a safety number.

## 6. Architecture-Blocking Decisions (Resolve BEFORE Stack Lock)

These cannot be retrofitted after topology lock — resolve them inside the first 30 days, in parallel with the spikes.

1. **The keystone topological-identity bet (Spike #1 result).** Generator-baked tags + geometric-signature backstop, OR — if Spike #1 kills it — C++ OCAF/TNaming in OCCT. This single result determines whether you are a Python-monolith shop or need a Rust/OCCT-FFI engineer (rarest, longest-lead hire). It sets the hiring plan and timeline; it must be the **first** thing you learn, not the last.

2. **AGPL-slicer file/subprocess isolation boundary.** Every FDM slicer (PrusaSlicer/libslic3r, CuraEngine, Orca) is AGPL-3.0; §13's network clause can compel open-sourcing the entire proprietary server. Lock the choice now: (a) **unmodified AGPL sidecar** — separate process/container/dependency-closure, STL-in/G-code-out over **files+CLI only**, no in-process `libslic3r` import, network-isolated (mere aggregation, §13 never reached); (b) commercial license; (c) from-scratch estimator. This dictates container topology and the build graph. Claude authors the license-scan CI gate (scancode/pip-licenses + SBOM diff that **fails the build** on any AGPL/GPL in the server's transitive closure or any in-process slicer import) and drafts the "separate work" memo **for counsel**; counsel signs the aggregation theory in writing. A demo-expedient in-process link already creates exposure.

3. **Hosted-Claude vs air-gapped-vLLM SKU seam.** Streaming controlled UAV geometry/intent to a hosted endpoint is a likely **deemed-export** (EAR 9A012/600-series, ITAR USML VIII); Anthropic ZDR does **not** cure it. Build ONE `LLMProvider` abstraction from commit #1 so no call site, log sink, or eval harness hard-codes Anthropic; add a CI lint failing on any Anthropic-SDK import outside the provider. This fork also sets the tenancy model (RLS soft-tenancy vs namespace/air-gap). File the jurisdiction/classification question (CJ/CCATS) with export counsel using the UAV example verbatim. (Not needed for the parts wedge itself, but the seam must exist before signing any controlled-data customer.)

4. **OCCT LGPL relink/notice (coupled to #3).** Pure SaaS never distributes the binary, so LGPL never triggers — but the on-prem/air-gapped SKU **is** distribution: the instant it ships, LGPL-2.1 §6 relink+notice attaches to OCCT (the OCCT "Open CASCADE Exception" relaxes some duties — counsel interprets). **Dynamic-link OCCT from the first on-prem build** (static linking makes relink unsatisfiable); gate the on-prem artifact on a signed "LGPL-cleared" counsel token.

**Non-delegable to Claude:** export classification, airworthiness/FS determination, ToS/EULA/NOTICE enforceability, the aggregation theory. Claude triages, flags, and drafts; counsel and a licensed PE decide. AI-drafted legal text is a first draft to accelerate counsel — never ship-ready.
