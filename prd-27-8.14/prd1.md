## Product Requirements Document 1: Core Foundation & Domain Architecture

### 1. System Overview

The platform is an AI-guided, generative hardware co-modeling workspace designed to compress the complex cycles of aerodynamic, mechanical, and manufacturing engineering into a fluid, conversational interface. Unlike traditional CAD tools that rely on manual boundary modeling or static feature trees, this system abstracts design intent into an **Event-Sourced Parametric Ledger**. The platform acts as an automated systems engineer, ensuring that physical objects generated by the underlying code kernels are inherently optimized for real-world constraints (e.g., flight physics, mass budgets, and manufacturing tolerances).

---

### 2. Domain Data Architecture

The internal representation of any project is treated as an **Agnostic Meta-Graph**. The system strictly decouples *design intent* from *mesh representation*.

```
                ┌──────────────────────────────────────┐
                │        Master Parametric Ledger       │
                │    (Single Source of Truth / JSON)   │
                └──────────────────┬───────────────────┘
                                   │
         ┌─────────────────────────┼─────────────────────────┐
         ▼                         ▼                         ▼
┌──────────────────┐      ┌──────────────────┐      ┌──────────────────┐
│  Aerodynamics    │      │    Mechanical    │      │  Manufacturing   │
│   Domain Node    │      │   Domain Node    │      │   Domain Node    │
└──────────────────┘      └──────────────────┘      └──────────────────┘

```

#### 2.1 The Master Parametric Ledger

The single source of truth for the entire vehicle state is a unified JSON schema. It stores structural dimensions, performance boundaries, and component states.

* **Dynamic Variables:** Properties computed automatically by the AI system loop (e.g., chord distribution, rib spacing, internal shell thickness).
* **Hard-Locked Variables (`HARD_LOCK`):** Properties explicitly input or frozen by the user (e.g., a specific wingspan value of $2800\text{ mm}$ or a customized 4.5mm pivot pin). These nodes are entirely immune to subsequent automated optimization cycles.

#### 2.2 Boundary Isolation & Computational Cascades

The architecture organizes components into distinct structural branches. When a localized node is mutated (e.g., modifying a landing gear bracket dimension), the system isolates the execution tree. It runs a localized re-compilation of that specific component branch, validating clearance envelope constraints without forcing a heavy, global re-computation of independent branches (e.g., the wingtip aero-lofts).

### 3. Structural Blueprinting Logic (`build123d`)

The geometric kernel relies entirely on programmatic generation via `build123d`. The AI generates, executes, and modifies procedural scripts to output deterministic 3D assets.

#### 3.1 Automated Slicing & Modular Assembly

Because structural components (such as a large aerodynamic wing) frequently exceed standard manufacturing bed volumes, the geometric compiler is injected with native modular slicing rules:

* **Sectional Splitting:** The pipeline tracks user-specified or hardware-limited build limits (e.g., $200\text{ mm}$ height increments) and programmatically cuts continuous meshes along normal planes.
* **Interlocking Joints:** At each partition plane, the script automatically lofts a lip-and-groove alignment edge along the outer shell profile and embeds alignment pins to ensure straight assembly alignment during post-processing.

#### 3.2 Automated Tolerance Padding

To compensate for standard manufacturing behavior (such as thermal plastic shrinkage or hardware variances), the `build123d` generator applies localized modifiers based on the target component database:

* **Slip-Fit Clearances:** Tracks cylindrical inserts (e.g., a 16mm carbon fiber wing spar) and applies a dynamic scaling padding (e.g., $+0.2\text{ mm}$) to the negative boolean cutout operation.
* **Fastener Insets:** Automatically embeds structural capture pockets for hardware nuts and bolt clearances directly into internal bulkheads based on standardized thread specifications.
